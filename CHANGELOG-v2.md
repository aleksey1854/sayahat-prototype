# Саяхат — версия 2

## Ревью адаптива (iPhone 14 Pro Max, 430px)
- Карта уезжала вбок. Было `min-width: 600px` + горизонтальный скролл внутри страницы: на телефоне правая половина схемы просто отрезалась. SVG с фиксированным viewBox перестроиться в колонку не умеет в принципе, поэтому карта переписана с SVG на HTML + CSS Grid. Десктопный вид сохранён один в один; на телефоне павильоны идут в колонку, зоны — 2×2, горизонтального скролла больше нет.
- Под футером главной висела пустая полоса. `body { padding-bottom: 74px }` стоял глобально в мобильном медиазапросе — это запас под фиксированную панель «Позвонить/WhatsApp», а она есть только на странице магазина. Перенёс запас внутрь самого CallBar: где панель есть, там и отступ.
- Контраст подписей на карте. Цвета переносил из SVG один в один, так что это не регрессия, но три пары не дотягивали до AA: подсказка на подсвеченном павильоне 2.58 → 4.54, «Главный вход» 3.42 → 4.84, стрелка входа 2.31 → 3.26 (порог для графики 3.0). Оттенки остались в янтарной гамме.

На твоё решение (не трогал):
- `--accent-700` (#AE6E10) используется как текст в `.eyebrow` по всему сайту: 3.81 на фоне сайта, 4.17 на белой карточке, 3.55 на тёплой панели — при норме 4.5 для мелкого текста. Затемнение на пару тонов почти незаметно и закрыло бы вопрос, но это бренд-цвет, поэтому решение за тобой.

## Визуальный апгрейд (по 3 репо: impeccable / emilkowalski / taste-skill)
Что взял (бренд не ломая):
- Моушн: все переходы переведены на ease-out-кривые (quint/expo), тактильный `:active` (лёгкий scale при нажатии, отклик мгновенный), плюс scroll-reveal.
- Reveal во всех браузерах: заменил CSS-only `animation-timeline` (только Chromium) на IntersectionObserver. Класс `.reveal`, скрытое состояние только при `html.js` — без JS/при сбое бандла контент виден, секция не «уедет пустой» (частая ловушка, о которой предупреждает impeccable). Карточки каталога появляются со стаггером по колонке; состояние в React-Set, поэтому фильтрация не вызывает мигания. На странице магазина reveal тоже подключён: секции (о магазине, ассортимент, промо, галерея, прайс, контакты, trust) — через лёгкий observer `RevealOnScroll` (страница статична, класс не слетит; hero и футер не трогаю).
- Скелетоны каталога вместо пустоты при загрузке (`app/loading.tsx`): скелетон-топбар той же высоты (без прыжка) + грид карточек-заглушек с шиммером. Спиннеров нет (leon/impeccable). Шиммер отключается при reduced-motion.
- Eyebrow-кикеры убраны со всех секций (главная, магазин, админ, кабинет). Осталось по одному осмысленному лейблу на страницу (бренд-строка над каталогом, подпись в hero магазина, шапки внутренних страниц) — это «голос», а не рефлекс. impeccable: кикер над каждой секцией = AI-почерк (55–95% генераций).
- Типографика: `text-wrap: balance` на заголовках, `pretty` в прозе, минусовой трекинг на крупном тексте (правило Apple).

Что сознательно НЕ трогал:
- Кремовый фон + Onest/Manrope + зелёный/янтарь — бренд Виктора. У impeccable прямое правило: для утверждённых бренд-цветов identity-preservation wins. Захочешь увести палитру от крема — сделаю отдельным вариантом, чтобы сравнить.
- Тире не менял: его банят англоязычные правила копирайтинга, в русском тире обязательно.
- Geist/Satoshi, холодные нейтрали, бенто, глассморфизм из taste-skill — это вкус под премиум-SaaS, у нас тёплый локальный рынок. Взял только универсальное.

## Ревью визуала (по чек-листу impeccable polish)
Проверил по коду (настоящий пиксель-просмотр — на твоей машине в браузере: в песочнице нет ни БД, ни headless-хрома). Что нашёл и сделал:
- Контраст: приглушённый текст держит 4.87–5.72 даже на тёплых панелях — главный флаг impeccable (серый на тёплом почти-белом) нас не касается, цвет подобран верно. `accent-700` (3.81) только на иконках, для графики порог 3:1 — проходит.
- Обёртка `<Reveal>` не ломает CSS: чувствительные селекторы (`:last-child`, `:first-of-type`) — про ячейки таблиц и строки внутри секций, соседство секций нигде не используется. Стики-хедер не внутри Reveal.
- Добавил `suppressHydrationWarning` на `<html>` — скрипт `js`-класса меняет разметку после серверного рендера (иначе React ругался бы, как с next-themes).
- Сбросил `transition-delay` на hover/active карточек — иначе тень запаздывала бы до 90мс (задержка нужна только для входа при скролле).
- Адаптив: все гриды схлопываются в 1 колонку; карта на мобильном — горизонтальный скролл с `min-width: 600px` (подписи читаемы); arenda и factbar переносятся; footer в колонку.

Осталось на заметку (не баги):
- `.cat-chip` ≈39px по высоте — проходит WCAG 2.2 AA (порог 24px), но меньше «комфортных» 44px. Для чипа нормально; скажешь — подниму паддинг на мобильном.
- CallBar (фикс. панель звонок/WhatsApp) сейчас только на страницах магазина. На главной снизу на мобильном остаётся ~74px запаса от общего правила. Вопрос к тебе/Виктору: нужен ли CallBar и на главной (тогда запас оправдан).
- Карточки каталога скрыты (opacity:0) до гидрации JS — обычный компромисс scroll-reveal. LCP это не бьёт: заголовок и хедер рисуются сразу, они не под reveal.
