# Свои фото в каталог

Сейчас в каталоге демо-обложки, нарезанные из 15 исходников. Скачать новые
из интернета я не могу: в песочнице открыты только github, npm и pypi —
Unsplash, Pexels и Wikimedia отдают `host_not_allowed`. А то, что лежит на
GitHub, это исследовательские датасеты с лицензией «только для науки» —
на коммерческий сайт клиента такое ставить нельзя.

Поэтому фото качаешь ты, а раскладывает скрипт.

---

## Порядок

**1. Скачай фото в папки по категориям**

Папки уже созданы: `photos-input/<категория>/`. Имена файлов любые,
размер любой, форматы jpg / png / webp / avif.

**2. Запусти**

```
npm run photos
```

Скрипт обрежет каждое фото по центру под пропорцию карточки (16:10),
уменьшит до 1000px, пережмёт в jpg и разложит по магазинам категории.
Маппинг `slug → файл` перезапишется автоматически.

В отчёте будет видно, где фото не хватает:

```
  meat      фото: 10 · магазинов: 10 · OK
  cloth     фото:  4 · магазинов: 14 · не хватает 10
```

Если фото меньше, чем магазинов, они пойдут по кругу — дубли будут,
но каталог не останется пустым.

**3. Задеплой, потом обнови базу**

```
git add . && git commit -m "photos" && git push
npm run covers -- --all
```

Порядок важен: фото это статика, они должны сначала уехать на прод.

## Что будет лучше стока

Сток остаётся стоком: чужой магазин, чужой город, часто чужая страна.
Для показа Виктору сойдёт, но перед запуском стоит заменить на настоящее.

Варианты по убыванию скорости:

- **Instagram арендаторов.** У тебя есть хендлы всех 50 — у большинства
  в ленте есть кадры прилавка. Разрешение спросить проще, чем кажется:
  им же выгодно.
- **2ГИС.** В карточке рынка есть фотографии от посетителей.
- **Сходить и отснять.** Час на рынке с телефоном закрывает вопрос
  полностью и навсегда: реальные прилавки, реальные продавцы, реальные
  вывески. Для сайта рынка это в разы сильнее любого стока — и это же
  аргумент в разговоре с Виктором про следующий этап.

Когда появятся настоящие фото — тот же `npm run photos`, скрипт всё
переразложит.
