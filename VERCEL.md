# Деплой на Vercel

Пошагово: от репозитория на GitHub до работающего сайта с базой и фото.
Проект уже подготовлен под Vercel — база на PostgreSQL, фото в Vercel Blob.

Порядок такой: сначала создаём базу и хранилище, наполняем базу локально,
потом деплоим. Так на сайте сразу будут магазины, а не пустой каталог.

## 0. Что нужно

- Аккаунт на vercel.com (вход через GitHub — проще всего).
- Проект уже запушен на GitHub (готово).

## 1. Импортировать проект

1. vercel.com → **Add New → Project** → выбрать свой GitHub-репозиторий → **Import**.
2. Framework Vercel определит сам (Next.js). Настройки сборки не трогай.
3. **Пока не нажимай Deploy** — сначала подключим базу и хранилище (шаги 2–3),
   иначе первая сборка упадёт без базы. Если уже нажал и упало — ничего
   страшного, задеплоим заново в конце.

## 2. Создать базу (Postgres)

1. В проекте на Vercel → вкладка **Storage** → **Create Database** → **Postgres**
   (в маркетплейсе может называться **Neon**) → создать, регион ближе к Казахстану
   (Frankfurt / eu-central).
2. Vercel сам добавит переменные окружения базы в проект. Нужны две строки
   подключения — их видно в настройках базы (**.env.local** / Quickstart):
   - строка с `-pooler` в адресе — это **DATABASE_URL**;
   - строка без `-pooler` (Direct / non-pooling) — это **DATABASE_URL_UNPOOLED**.
3. Проект → **Settings → Environment Variables**. Убедись, что есть
   `DATABASE_URL` и `DATABASE_URL_UNPOOLED`. Если Vercel создал переменные
   с другими именами (например `POSTGRES_PRISMA_URL`, `POSTGRES_URL_NON_POOLING`) —
   просто добавь `DATABASE_URL` = значение pooled-строки и
   `DATABASE_URL_UNPOOLED` = значение non-pooling-строки.

## 3. Создать хранилище фото (Blob)

1. **Storage → Create → Blob** → создать.
2. Vercel добавит переменную `BLOB_READ_WRITE_TOKEN` в проект автоматически.
   Проверь в Settings → Environment Variables, что она появилась.

## 4. Добавить остальные переменные

Проект → **Settings → Environment Variables** → добавь:

- `SITE_URL` — адрес сайта. Сначала можно временный вида
  `https://ИМЯ-ПРОЕКТА.vercel.app`, после подключения домена поменяешь.
- `SESSION_SECRET` — длинная случайная строка. Сгенерировать:
  на Linux/Mac `openssl rand -base64 32`, либо любой генератор паролей на 40+ символов.
- (по желанию) `NEXT_PUBLIC_YM_ID`, `NEXT_PUBLIC_GA_ID`,
  `GOOGLE_SITE_VERIFICATION`, `YANDEX_VERIFICATION`.

## 5. Наполнить базу (локально, но в облачную базу)

Чтобы на сайте были магазины, создаём таблицы и заносим данные со своего
компьютера — писать они будут сразу в облачную базу Vercel.

1. Скопируй строки подключения из шага 2 в локальный файл `.env`:
   ```
   DATABASE_URL="postgresql://…-pooler…?sslmode=require"
   DATABASE_URL_UNPOOLED="postgresql://…?sslmode=require"
   ```
2. В терминале проекта:
   ```bash
   npm install
   npx prisma db push        # создаёт таблицы в облачной базе
   ```
3. Наполнение:
   - **для теста** можно залить демо: `npm run db:seed` (6 категорий, 9 магазинов,
     аккаунты admin/dastarkhan). Реальными данными наполняй через `/admin`.
   - **боевой вариант** — сразу заводи реальные магазины через `/admin`
     (см. README). `db:seed` реальные данные стирает — не запускай его после.

Фото при локальном наполнении: если в локальном `.env` не задан
`BLOB_READ_WRITE_TOKEN`, фото пишутся на диск и на Vercel их не будет.
Чтобы фото сразу летели в облако, добавь в локальный `.env` и токен Blob
(значение из Settings → Environment Variables на Vercel) — тогда загрузка
из локального `/admin` кладёт фото в тот же Blob, что видит сайт.

## 6. Задеплоить

Вкладка **Deployments → Redeploy** (или сделай любой `git push` — Vercel
пересоберёт сам). Через пару минут сайт открывается по адресу
`https://ИМЯ-ПРОЕКТА.vercel.app`.

Проверь: `/` — каталог с магазинами, `/login` под admin, загрузка фото в кабинете.

## 7. Свой домен

Проект → **Settings → Domains** → добавить домен → у регистратора прописать
записи, которые покажет Vercel (обычно A-запись или CNAME). SSL Vercel
выпустит сам. После этого поменяй `SITE_URL` на боевой домен и сделай Redeploy.

## Как обновлять сайт дальше

Просто `git push` в GitHub — Vercel пересобирает и выкатывает автоматически.
База и фото при этом не трогаются (они снаружи, в Postgres и Blob).

Если менялась схема базы (`prisma/schema.prisma`) — после пуша один раз
локально прогнать `npx prisma db push` на облачную базу.

## Если что-то пошло не так

- Сборка упала на базе → проверь `DATABASE_URL` и `DATABASE_URL_UNPOOLED`
  в Environment Variables (обе, правильные строки).
- Сайт открылся, но каталог пустой → не выполнен шаг 5 (`db push` + наполнение).
- Фото не грузятся / пропали после деплоя → не создан Blob (шаг 3) или в
  окружении нет `BLOB_READ_WRITE_TOKEN`.
- Логи ошибок: проект → Deployments → конкретный деплой → **Runtime Logs**.

## Границы бесплатного тарифа

Free-план Vercel рассчитан на некоммерческие проекты и имеет лимиты (трафик,
серверное время). Для теста Этапа 1 хватает с запасом. При росте посещаемости
или как коммерческий проект — платный план (~$20/мес) либо переезд на VPS
(инструкция в DEPLOY.md, код это поддерживает).
